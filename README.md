# 브랜드 허브

내가 기획한 브랜드(개요 / 로고·이미지 / 발표자료 / 브랜드컬러 / 카피문구 / 메뉴라인업 / 시즌메뉴 / 이벤트 / 코스트 계산기)를
한 곳에서 정리하는 정적 웹사이트입니다. GitHub Pages로 호스팅하고, 브랜드를 등록/수정하면
사이트에서 바로 GitHub 저장소에 커밋됩니다. 이미지와 PPT도 URL 없이 파일 선택만으로 업로드됩니다.

## 폴더 구조

```
(저장소 루트)/
├─ index.html            브랜드 목록(홈)
├─ brand.html            브랜드 상세 페이지 (탭: 개요/로고·이미지/발표자료/컬러/카피/메뉴/시즌메뉴/이벤트/코스트계산기)
├─ add-brand.html        브랜드 등록/수정 폼 (카테고리별 탭 구조)
├─ settings.html         GitHub 연결 설정
├─ assets/
│  ├─ css/style.css
│  └─ js/ (index.js, brand.js, add-brand.js, settings.js, github-api.js)
├─ data/
│  └─ brands/
│     ├─ index.json      전체 브랜드 목록(요약 정보)
│     └─ jaengban-jip.json  샘플 브랜드 데이터(쟁반집)
├─ images/
│  └─ jaengban-jip/       브랜드별 이미지 저장 폴더 (업로드 시 자동 생성됨)
└─ files/
   └─ jaengban-jip/       브랜드별 PPT 등 발표자료 저장 폴더 (업로드 시 자동 생성됨)
```

## GitHub Pages로 배포하기

1. 이 폴더 안의 내용물을 GitHub 저장소 루트에 올립니다. (`index.html`이 저장소 최상위에 오도록.)
2. 저장소의 **Settings → Pages** 로 이동합니다.
3. **Source**를 `Deploy from a branch`로, 브랜치는 `main`, 폴더는 `/root`로 설정합니다.
4. 몇 분 후 `https://아이디.github.io/저장소이름/` 주소로 사이트가 보입니다.

## GitHub 연결 설정 (최초 1회)

브랜드를 등록/수정/파일 업로드하면 사이트가 자동으로 저장소에 커밋하도록 하려면, 먼저 개인 액세스 토큰을 연결해야 합니다.

1. 사이트에서 **⚙ GitHub 연결** 버튼을 눌러 `settings.html`로 이동합니다.
2. [github.com/settings/tokens](https://github.com/settings/tokens?type=beta) 에서 "Generate new token"으로 Fine-grained token을 만듭니다.
   - Repository access → Only select repositories → 이 사이트가 올라간 저장소 선택
   - Permissions → Repository permissions 목록에서 **Contents**를 찾아 "Read and write"로 설정 (기본값 "No access"라서 꼭 바꿔야 합니다)
3. 생성된 토큰을 복사해서 설정 화면의 아이디 / 저장소 이름 / 브랜치 / 토큰 입력칸에 넣고 "연결 테스트" → "저장하기"를 누릅니다.

⚠️ 토큰은 사용 중인 브라우저의 localStorage에만 저장됩니다. 본인만 쓰는 컴퓨터에서 사용하세요.

## 브랜드 등록 / 수정하는 방법

1. **+ 새 브랜드 추가**를 누르면 카테고리별 탭(기본정보 / 로고·이미지 / 발표자료 / 개요 / 브랜드컬러 / 카피문구 / 점심메뉴 / 저녁메뉴 / 시즌메뉴 / 이벤트)으로 나뉜 폼이 나옵니다. 스크롤을 길게 내릴 필요 없이 탭을 눌러 이동하면 됩니다.
2. 기본정보 탭에서 브랜드명을 입력하면 슬러그가 자동으로 채워집니다. **이미지나 PPT를 업로드하려면 슬러그가 먼저 정해져 있어야 합니다.**
3. 각 항목을 채우고, 맨 아래 **브랜드 등록하기** 버튼을 누르면 지금까지 입력한 모든 탭의 내용이 한 번에 GitHub 저장소에 커밋됩니다.
4. 저장 직후 브랜드 상세 페이지로 이동하며, 방금 저장한 내용을 바로 보여줍니다. 실제 배포된 사이트(다른 브라우저/기기)에는 GitHub Pages가 다시 빌드되는 몇 초~1분 정도 후 반영됩니다.
5. 이미 등록한 브랜드는 상세 페이지의 **✏️ 수정하기** 버튼으로 들어가 수정하고 저장하면 같은 방식으로 반영됩니다. (수정 화면에서는 슬러그를 바꿀 수 없습니다.)

## 이미지 업로드 (로고 / 브랜드 이미지 / 메뉴 사진)

- **로고 · 이미지** 탭에서 브랜드 로고 1장과, 매장 사진·컨셉 이미지 같은 "브랜드 이미지"를 여러 장 등록할 수 있습니다.
- 점심·저녁 메뉴, 시즌메뉴 각 항목에도 사진을 넣을 수 있습니다.
- **"파일 선택"** 버튼으로 바로 업로드하면 `images/브랜드슬러그/` 폴더에 자동 저장되고 경로도 자동으로 채워집니다. 외부 이미지 URL을 텍스트 칸에 직접 입력해도 됩니다.
- 이미지는 5MB 이하까지 업로드할 수 있습니다.
- 방금 올린 이미지는 GitHub Pages가 재배포될 때까지(보통 몇 초~1~2분) 잠깐 안 보일 수 있는데, 브랜드 상세 페이지가 자동으로 몇 차례 재시도하니 잠시 기다리면 나타납니다.

## 발표자료(PPT) 업로드 & 다운로드

- **발표자료** 탭에서 사업계획서, 브랜딩 PPT 같은 파일을 등록할 수 있습니다. 자료명을 적고 "파일 선택"으로 `.ppt` 또는 `.pptx` 파일을 올리면 `files/브랜드슬러그/` 폴더에 자동 저장됩니다.
- 파일은 25MB 이하까지 업로드할 수 있습니다.
- 등록하면 브랜드 상세 페이지의 **발표자료** 탭에 다운로드 버튼이 생깁니다. 사이트 주소만 알면 로그인이나 별도 프로그램 없이 누구나(다른 컴퓨터, 다른 사람) 그 버튼을 눌러 PPT를 바로 다운로드할 수 있습니다.
- 여러 개의 자료를 올려두고 각각 이름을 다르게 붙여서 관리할 수 있습니다 (예: "1차 기획안", "최종 사업계획서").

GitHub 연결이 안 되어 있거나 문제가 생기면, 등록 화면 하단의 "고급: JSON 파일로 직접 내보내기"에서 JSON을 다운로드해 수동으로 `data/brands/`에 넣는 방법도 여전히 사용할 수 있습니다. (이 경우 이미지·PPT 파일은 직접 저장소에 올리고 경로를 입력해야 합니다.)

## 코스트 계산기

브랜드 상세 페이지의 "코스트 계산기" 탭은 두 부분으로 구성됩니다.

- **메뉴 원가표**: 브랜드 JSON에 등록된 모든 메뉴(점심+저녁)의 판매가/원가를 기준으로
  원가율(원가÷판매가×100), 마진액(판매가-원가), 마진율을 자동 계산합니다.
- **빠른 계산기**: 판매금액과 원가액만 입력하면 원가율/마진액/마진율을 즉시 계산해주는 도구이며,
  직접 항목을 추가해 임시로 비교해볼 수 있습니다. (이 임시 항목은 브라우저에만 저장되며 JSON 파일에는 반영되지 않습니다.)
